Angular のサービスは、 Web アプリケーションに共通の、特定のタスクを実行するシングルトンです。 たとえば、 $http サービス は、ブラウザの XMLHttpRequest オブジェクトに対する低レベルアクセスを提供します。

Angular のサービスを使用するには、そのサービスに依存する側 (コントローラや別のサービス) の依存関係として、 Angular のサービスを指定します。あとは Angular の依存性注入のサブシステムが面倒を見てくれます。 Angular の依存性注入サブシステムは、サービスのインスタンス化、依存関係の解決、 そして要求どおりにファクトリ関数へ依存サービスを供給することを担当します。

Angular は「コンストラクタ」注入を使って、依存性を注入します (サービスはファクトリ関数を通じて渡されます)。 JavaScript は動的型付け言語なので、 Angular の依存性注入サブシステムは、 静的型を使ってサービスの依存性を特定することができません。 そのため、下記のように、依存する側はその依存関係を $inject プロパティを使って明示的に定義しなければなりません。

    myController.$inject = ['$location'];

Angular Web フレームワークは共通の処理のためのサービスセットを用意しています。 他の Angular コアの変数や識別子のように、組み込みのサービスは必ず $ で始まります (先述の $http サービスと同様です)。 独自のカスタムサービスを作成することもできます。
