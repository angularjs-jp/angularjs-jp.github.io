    "Nothing here yet!" と表示されたHTMLページは下記のコードで構成されています。 このコー​​ドには、今後必要になる、いくつかの重要なAngularの要素が含まれています。

    app/index.html: 

    <!doctype html>
<html lang="en" ng-app>
<head>
  <meta charset="utf-8">
  <title>My HTML File</title>
  <link rel="stylesheet" href="css/app.css">
  <link rel="stylesheet" href="css/bootstrap.css">
  <script src="lib/angular/angular.js"></script>
</head>
<body>

  <p>Nothing here {{'yet' + '!'}}</p>

</body>
</html>

    このコードがやっていること

    • ng-app ディレクティブ:

        <html ng-app>

      ng-app 属性はAngular directive を表しています (ディレクティブの名前はngApp 、Angularでは属性名にname-with-dashes を、対応するディレクティブ名にcamelCaseを使用します) 。これは、Angularがアプリケーションのルート要素であると見なす要素にフラグを設定するために使用されます。これにより、アプリケーション開発者は、Angularアプリケーションとして扱われる部分をHTMLページ全体にするかもしくはその一部だけにするかを選ぶことができます。

    • AngularJS scriptタグ:

        <script src="lib/angular/angular.js">

      このコードはangular.jsスクリプトをダウンロードし、HTMLが完全にダウンロードされたときにブラウザで実行される callbackを登録します。コールバックが実行されると、AngularはngAppディレクティブを探します。AngularがngAppディレクティブを見つけた場合、ngAppディレクティブが定義された要素をアプリケーションDOMのルートとして起動します。

    • 2重波括弧バインディングの中に式が入っています:

        Nothing here {{'yet' + '!'}}

      この行は、テンプレート機能の基本的な特徴を示しています。2重波括弧{{ }}で示されるバインディングや'yet' + '!'のようなシンプルな式がこのバインディングで使われています。

      このバインディングはAngularに式を評価しバインディングされたDOMに結果を挿入するように伝えます。これは一度だけの挿入ではなく、次のステップで説明するように、式評価の結果が変わるたびに実行される継続的な更新になります。

      期待通りにこのテンプレートはAngularによって処理され、HTMLページには次のテキストが表示されます： "Nothing here yet!"

    AngularJSアプリを起動する

    自動的にAngularJSアプリを起動するngAppディレクティブを使用することは非常に簡単で、ほとんどの場合に適した選択肢です。スクリプトローダーを使用しているときなど、高度なケースでは、アプリを起動するためにimperative / manual wayを使用することができます。

    アプリの起動中に発生する3つの重要な事があります:

    1. アプリ内のDI (Dependency Injection)に使用されるinjectorが作成されます。

    2. 次にinjectorはアプリケーションのモデルのコンテキストになるルートスコープを作成します。

    3. 次にAngularはngAppルート要素から始まり、途中で見つけたディレクティブやバインディングを処理しDOMをレンダリングします。

    アプリケーションが起動されると、モデルを変更する可能性のあるブラウザイベント (マウスクリック、キー入力や着信HTTP応答など) を待ち受けます。一度そのようなイベントが発生したら、Angularはそれがモデルの変化を引き起こしたかどうかを検出し、変更が検出された場合、影響を受けるすべてのバインディングを更新することにより、ビューに反映します。

    アプリケーションの構造は今のところ非常にシンプルです。テンプレートには、1ディレクティブと1つの静的バインディングが含まれており、モデルは空です。

    作業ディレクトリ内のすべてのファイルについて

    作業ディレクトリ内のファイルのほとんどは典型的なAngularプロジェクトの立ち上げに使われるangular-seed projectプロジェクトから来たものです。angular-seedプロジェクトは最新のAngularライブラリ、テストライブラリ、スクリプト及びシンプルなサンプルアプリケーションを含んでいます。またすべて典型的なwebアプリを開発するために事前設定済みです。
